## Определение номера менеджера по продажам

## 1. Цель
Разработать AGI-программу на NodeJS для Asterisk, предназначенную для определения номера менеджера по продажам на основе данных CDR.
Программа должна проверять, осуществлял ли менеджер по продажам исходящий звонок клиенту в течение последних 60 минут, и в случае обнаружения такого звонка сохранять номер менеджера в переменную канала `SALES_PHONE`.
Если подходящий менеджер не найден, переменная `SALES_PHONE` должна быть установлена в значение `0`.

---

### 2. Входные данные

**Номер клиента**

Источник по умолчанию — `CALLERID(num)`.

Перед использованием номер должен быть приведён к нормализованному виду.

---

### 3. Выходные данные

Результатом работы AGI-программы является установка переменной канала:

- `SALES_PHONE = <номер менеджера>` — если найден менеджер, соответствующий условиям поиска;
- `SALES_PHONE = 0` — если подходящий менеджер не найден.

Возврат значений через `stdout` не используется.  
Единственным результатом считается корректно установленная переменная канала.

---

### 4. Функциональные требования

#### 4.1. Логика поиска

AGI-программа должна выполнять следующую последовательность действий:

1. Получить номер клиента из входных данных.
2. Выполнить поиск записей в CDR за последние 60 минут относительно текущего времени выполнения AGI.
3. Учитывать только записи, удовлетворяющие всем следующим условиям:
    - звонок является исходящим;
    - номер менеджера по продажам указан в A-стороне звонка (поле `src`);
    - номер клиента указан в B-стороне звонка (поле `dst`);
    - для определения времени звонка используется время его начала (например, `calldate` / `start`).
4. Отсортировать найденные записи по времени звонка в порядке убывания.
5. Использовать запись с самым поздним временем звонка.
6. Извлечь номер менеджера и установить его в переменную `SALES_PHONE`.

Если записи, удовлетворяющие указанным условиям, отсутствуют, программа должна установить `SALES_PHONE = 0`.

---

#### 4.2. Нормализация номеров

Для корректного и однозначного сопоставления телефонных номеров необходимо:

- удалять пробелы, скобки, дефисы и символ `+`;
- приводить номер к единому цифровому формату;
- приводить номера вида `8XXXXXXXXXX` к формату `7XXXXXXXXXX`;
- применять одинаковую логику нормализации как к номеру клиента, так и к номерам, хранящимся в CDR.

Короткие внутренние номера не должны рассматриваться как номера клиентов и не участвуют в поиске.

---

#### 4.3. Источник данных

Источником данных для поиска является CDR Asterisk, размещённый в SQL-базе данных.

Параметры подключения к базе данных должны задаваться через переменные окружения или внешний конфигурационный файл.

---

### 5. Нефункциональные требования

#### 5.1. Надёжность

При любой ошибке выполнения (ошибка подключения к базе данных, ошибка SQL-запроса, ошибка обработки данных) AGI-программа должна корректно завершаться и в обязательном порядке устанавливать `SALES_PHONE = 0`.

---

#### 5.2. Логирование

Программа должна фиксировать ключевые этапы и результаты своей работы, включая:

- исходный номер клиента и его нормализованное представление;
- факт наличия либо отсутствия подходящих записей в CDR;
- найденный номер менеджера либо установленное значение `0`;
- информацию об ошибке при её возникновении.
